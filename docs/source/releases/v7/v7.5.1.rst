Version 7.5.1
=============

Hotfix for v7.5.0: the production Docker image still pinned
``python:3.10-slim``, but the NN metamodel introduced in v7.5.0 imports
``typing.Self`` which was added in Python 3.11 (PEP 673). The backend
container failed to start on the EC2 deploy with
``ImportError: cannot import name 'Self' from 'typing'``. The CI matrix
already runs on 3.11 and 3.12, so the issue only surfaced at deploy time.

Fixes
-----

* Bumped the runtime Docker base image from ``python:3.10-slim`` to ``python:3.12-slim`` so the production image matches the CI matrix and ``setup.cfg``'s declared ``python_requires = >=3.11``. No application code changes — the NN ``Self`` import works as-is on 3.12.
