◐What is not tested

Whether the generated app boots

Every layer asserts generation produced the right output — not that the app runs. That fidelity check (import-smoke + start the backend, hit /openapi.json) is the deferred Phase-3 boot check. It’s out of scope on purpose: free-model output often doesn’t boot yet, so a boot assertion would be a known-flaky gate. When it lands it becomes both a fix-loop input and the assertion these tests are missing.

◆CI & the one caveat

Layers 1–3 gate every PR. Layers 4–5 are nightly / manual.

The deterministic layers are your dependable everyday coverage. The full live browser test drives the real classifier and shared GPU, so it will occasionally go red for infra reasons (GPU load, a classifier hiccup) unrelated to any code change — treat it as a fidelity signal, not a blocking gate.

One thing to protect: the “Use-the-free-model does nothing” race is production-build-only (it doesn’t reproduce in dev or jsdom) and is held by the startingRunRef guard in SmartGenByokDialog — keep that guard; no automated test can catch a regression of it.
